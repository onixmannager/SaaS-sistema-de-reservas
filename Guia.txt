PASOS PARA PONER EN PRODUCCIÓN
===============================

PASO 1 — Crear proyecto Firebase
---------------------------------
1. Ve a https://console.firebase.google.com
2. Clic en "Añadir proyecto" y ponle nombre (ej: mi-agenda)
3. Ve a Firestore Database → Crear base de datos → modo producción
4. Ve al engranaje ⚙️ → Configuración del proyecto → icono </> (Web)
5. Registra la app y copia las 6 credenciales que aparecen:
   apiKey, authDomain, projectId, storageBucket, messagingSenderId, appId


PASO 2 — Reglas de Firestore
------------------------------
En Firebase ve a Firestore → Reglas y pega esto:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /reservations/{id} {
      allow create: if true;
      allow read, update, delete: if false;
    }
  }
}

Clic en Publicar.


PASO 3 — Desplegar en Vercel
------------------------------
1. Ve a https://vercel.com e inicia sesión con GitHub
2. Clic en "Add New Project"
3. Selecciona tu repositorio de GitHub
4. No cambies nada → clic en "Deploy"


PASO 4 — Variables de entorno
------------------------------
1. En Vercel ve a Settings → Environment Variables
2. Añade estas 7 variables con los valores de Firebase:

   FIREBASE_API_KEY
   FIREBASE_AUTH_DOMAIN
   FIREBASE_PROJECT_ID
   FIREBASE_STORAGE_BUCKET
   FIREBASE_MESSAGING_SENDER_ID
   FIREBASE_APP_ID
   ADMIN_PASSWORD        ← la contraseña que quieras para el panel admin

3. Ve a Deployments → tres puntos ... → Redeploy
